## Findings considered and deferred/rejected

Recorded so nobody re-audits them:

- **Backend static type checking (mypy/pyright ratchet)**: real gap (frontend
  is strict-typed, the ~24k-line backend has no checker), deferred — adopt as
  a ratcheted CI step after 001 lands rather than bundling it there.
- **Commit → image → deploy pipeline + backup/DR runbook**: deferred pending
  a hosting-target decision by the maintainer; see plan 005 maintenance
  notes. The production Dockerfiles are ready; the automation and the
  Postgres PITR/restore drill are the missing halves.
- **Agent-behavior regression harness** (distinct from roadmap plan 045's
  retrieval eval): nothing verifies model-driven behavior (tool selection,
  approval triggering, delegation routing) since live LLM calls are blocked
  in tests. Worth a design spike with pydantic-ai test doubles + golden
  transcripts; not planned here because it is design-heavy and the maintainer
  should scope which behaviors are release-blocking.
- **Skills document download audit gap**: same egress-audit gap as plan 002
  Step 6 exists in `services/skills/documents/create_document_download.py`
  and `get_document_markdown.py`; follow plan 002's pattern when picked up.
- **`storage_object_response` buffers whole objects for non-local providers**
  (`services/storage/utils.py`): latent only — production downloads use
  direct signed URLs. Revisit if app-proxied cloud serving is ever enabled;
  plan 002's `stream_object` is the primitive to reuse.
- **SSE parser throwing on unknown event names**: deliberate (breaks stale
  clients on purpose; ship client changes first) — not a bug, do not "fix".
- **Files soft storage limit not enforced at upload**: by design — it is a
  soft counter (`get_files_usage` docstring), not a quota.
- **`security_events` lacking a workspace column**: settled decision
  (super-admin-only surface in v1, per `docs/plans/000_README.md`).
